SCENARIO 01 — LOGIN
════════════════════════════════════════════════════════════════════════════════

website: https://the-internet.herokuapp.com/login

Normal locators:
{
  "username_input": "#username",
  "password_input": "#password",
  "login_button":   "button[type='submit']"
}

Broken locators for healing demo:
  "login_button":   "button[type='submit999']"          ← triggers S4 (Role-Based)
  "username_input": "#username_BROKEN"                  ← triggers S7 (ID/Name)
  "password_input": "input[type='password_BROKEN']"     ← triggers S9 (CSS combo)

Expected healing path:
  login_button   → S4 get_by_role('button', name='Login')
  username_input → S7 #username  /  S5 get_by_label('Username')
  password_input → S9 input[type='password']

         (or any page with an HTML5 <video> element)

Broken locators for healing demo:

  page.evaluate("document.querySelector('video').play()")
  page.evaluate("document.querySelector('video').pause()")

Additional media element locators (for future scenarios):
{
  "audio_player":      "audio",                   ← HTML5 <audio>
  "audio_play_button": "audio[controls]"

  "canvas_element":    "canvas",                  ← HTML5 <canvas>
  "canvas_area":       "#canvas"

  "svg_circle":        "svg circle"
}

Broken versions for healing:
  "audio_player":      "audio#BROKEN"    → S9 audio / S11 //audio[1]
  "canvas_element":    "#canvas-BROKEN"  → S9 canvas / S11 //canvas[1]
  "svg_element":       "svg#BROKEN"      → S9 svg    / S11 //svg[1]

SCENARIO 13 — GEOLOCATION
════════════════════════════════════════════════════════════════════════════════

website: https://the-internet.herokuapp.com/geolocation

Normal locators:
{
  "where_am_i_button": "button[onclick*='getLocation'], .example button",
  "lat_result":        "#lat-value",
  "long_result":       "#long-value"
}

Broken locators for healing demo:
  "where_am_i_button": "button#BROKEN_WHERE_AM_I"   ← triggers S6 (text: 'Where Am I')
  "lat_result":        "#lat-value-BROKEN"           ← triggers S7 (ID: #lat-value)
  "long_result":       "#long-value-BROKEN"          ← triggers S7 (ID: #long-value)

Expected healing path:
  where_am_i_button → S6 get_by_text('Where Am I', exact=False)
                    → S4 get_by_role('button', name='Where Am I')
  lat_result        → S7 #lat-value
  long_result       → S7 #long-value

Browser context setup required:
  context.set_geolocation({"latitude": 25.3176, "longitude": 82.9739})
  context.grant_permissions(["geolocation"])


════════════════════════════════════════════════════════════════════════════════
  QUICK HEALING STRATEGY REFERENCE
════════════════════════════════════════════════════════════════════════════════

  S1  — Stored / Previously Healed Locators     (locator_store.json)
  S2  — data-testid / data-automation attrs
  S3  — ARIA attributes (aria-label, aria-labelledby)
  S4  — Role-based locators (get_by_role)
  S5  — Accessibility label / <label> text (get_by_label)
  S6  — Visible text locators (get_by_text / text=)
  S7  — ID or name attributes
  S8  — Element type + partial text (XPath contains)
  S9  — Composite CSS selectors (tag + class + attr)
  S10 — Neighbour / relative locators
  S11 — XPath with hierarchical heuristics
  S12 — Anchor to stable container context
  S13 — AI Healer (OpenAI GPT-4o) — last resort
